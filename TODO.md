# Справочник категорий

Реализовать справочник категорий - просмотр всех записей, добавление, изменение и удаление записей.

Макеты: https://www.figma.com/design/KCYaDH4HaImmOB7ZASq784/Zidium?node-id=1508-21270&t=ibK1mdwpvUswwdBb-1
Макеты обощённые для всех справочников, у каждого конкретного справочника свои поля.

## Backend

Swagger: https://zidium3-backend.zidium.net/swagger/index.html?urls.primaryName=Front
Url: https://zidium3-backend.zidium.net/

Api импортировать с помощью пакета openapitools/openapi-generator-cli.

## Просмотр всего справочника

Url в браузере: /categories

Макет "Справочник".

Заголовок - "Categories".

Поля: 
- "Id"
- "Name"

Для получения данных выполнять запрос:
GET /front/categories

Изначально запрашивать страницу с pageNumber = 0.

При скролле до её конца (или если ещё осталось место на странице) запрашивать следующую, и так пока не вернётся количество строк меньше, чем размер страницы.

При изменении текста в строке поиска передавать его в параметре search.

При клике по названию колонки Name нужно переключать направление сортировки (sortDesc, изначально false).

Если поле canAdd = false, то все элементы изменения справочника должны быть скрыты.

## Добавление записи

Для добавления записи вызывать метод:
POST /front/categories

## Редактирование или просмотр записи

Url в браузере: /categories/{id}

Для получения данных записи вызывать метод:
GET /front/categories/{id}

Поля: 
- "Id" (текст, не input)
- "Name"

Если у записи canEdit = false, то все поля становятся readonly, кнопка сохранения скрывается.

Для поля Name нужна асинхронная валидация на стороне сервера. Вызывать метод:
GET /front/categories/name-exists

Параметр id - id текущей записи, или null при добавлении.
Если метод вернул true, то название уже занято, и поле должно считаться невалидным.

Для сохранения данных вызывать метод:
POST /front/categories/{id}

## Удаление записи

Если у записи canDelete = false, то удаление недоступно, кнопка удаления скрыта.

Перед удалением показать диалог подтверждения.

Для удаления записи вызывать метод:
DELETE /front/categories/{id}

## Вход в систему

Все ранее указанные методы backend требуют передачи JWT-токена в заголовке Authorization.

Макеты для входа в систему: 
- "Авторизация"
- "Авторизация / Ошибки / Не заполнено"
- "Авторизация / Ошибки / Ошибка входа"

Для получения токена вызывать метод:
POST /front/logon

Для обновления токена вызывать метод:
POST /front/logon/refresh-token
